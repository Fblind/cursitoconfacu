Comentarios
  • Código no evaluado ni ejecutado por el runtime.
  • Usarlo poco, a conciencia y temporalmente si se necesita.
  • Código y tests muestran utilización del sistema.
  • De una línea o Multi línea.

Valores
  • Representaciones de algo, Ej: $0 o gratis, 2*1 o 2 al 50%, venis? si o no
  • Pueden ser números, texto, funciones, listas, etc
  • Si los incluimos directamente en el codigo los llamamos "literales"
  • En JavaScript los valores siempre estan asociados a un "tipo de dato"
Tipos de datos

Debilmente tipado

  • Variables no estan asociadas a un tipo de dato
  • Pueden "mutar" es pseudo dinamico y toma el valor mediante la asignación
  • Representacion de tipos pequeña y abarcativa

Tipos del lenguaje

  • Number (NaN)
  • BigInt
  • String
  • Boolean
  • Object (Function, Array, Date)
  • null
  • undefined
  • Symbol
Sacando object, los demás se llaman primitivas

typeof operator

Representación en texto del tipo de dato de un valor

            
              var a; typeof a;                    // "undefined"
              a = "hello world"; typeof a;        // "string"
              a = 42; typeof a;                   // "number"
              a = true; typeof a;                 // "boolean"
              a = null; typeof a;                 // "object" -- weird, bug
              a = undefined; typeof a;            // "undefined"
              a = { b: "c" }; typeof a;           // "object"
              a = Symbol(); typeof a;             // "symbol"

Coercion

  • Convertir valores implicitamente
  • Convergen en una primitiva
  • La conversion se hace en tiempo de ejecucion

Type casting es explicito y se hace en tiempo de compilacion

            
              var a = 42;
              var b = a + "";       // "42" implicit coercion
              var c = String( a );  // "42" explicit coercion (casting)
              var d = "42"
              var e = +d            // 42 implicit

Variables
  • Cajas para guardar valores o referencias
  • Nombres representativos para usarlo en nuestro código
  • Keyword más nombre empezando con a-z, A-Z, $, o _ y después lo que querramos más 0-9
  • camelCase
  • Declaración, inicialización (asignacion), uso, * re-asignación

Declaración

  • var
  • let
  • const
  • * sin keyword

var

  • Function scoped
  • Se puede re-asignar
  • Disponible "antes de asignarla" (hoisting)

Hoisting

Llevar para arriba la declaración dentro del scope (las functions declarations tmb!)

let

  • Block scoped
  • Se puede re-asignar
  • Disponible despues de declararlo (TDZ)

const

  • Block scoped
  • NO se puede re-asignar
  • Disponible despues de declararlo (TDZ)
